1. How can I pass data from PARENT component to CHILD component?

All right let's get into some actual code. Passing data from one component to another is extremely important. We will start with parent to child communication. Remember a child component means its selector is inside of the parents template.

Quick Note: I am using inline templates because its easier to show the examples but for larger more complex projects Angular recommends seperating the view and styles

We actually have 2 different ways to handle such input:

We can use the Input decorator or the input function

2. @Input decorator

To do this we need to work with the parent and child component class and their respective templates. The steps themselves are not very complex but as a beginner I really struggled with this. Let's look at the steps.

  • Child Component Class- import the Input decorator from angular core.
  • Child Component Class- create a class property/member 'decorated' with the Input. This is where our data will be stored. In other words, this is the property that we will eventually be able to display in our template.
  • Child Component Template- add the class property to your template with text interpolation. The double curly braces syntax. Which will be covered later.
  • Parent Component Class- import the child component into the parents class
  • Parent Component Class- create another class property/member that will hold the value we want to pass down to our child.
  • Parent Component Template- add the child component selector to parents template then use the class property you created in the child to then bind it to the property you just created inside your parent.
  • And that is it! Not too bad right? Actually, its rather straightforward but when I first started learning Angular I could not seem to grasp this. I hope it helps others that felt the same way.

3. The input function

This new approach is based on signals. Signals will be discussed in detail later. For now, lets just see the small differences in how to use it.

  • Differences: there are differences in how we define the import, how we define the class member, and how we use the value in the template.
  • Explanation: as you can see below we use lowercase input instead of Input. We use the input function instead of the decorator. Finally, we since the value is a signal we call it with parentheses inside of the template. Otherwise, all the other steps are the same.

4. What else do I need to know about input communication?

There are a few more interesting thing to know about using passing data with the input decorator/function. Let's look at some of them.

  • Required inputs: using a config object you can set the input to be mandatory. This means an error will be reported at run time.
  • Input transformations: allow you to specify a function to modify the value of an input. This can help with ensuring the input adheres to a specific format or type. These functions must be pure and statically analyzable at build time. The example below takes a number input property but the parent may provide it as a string so we use the transform function to make sure the value is always a number. Angular actually provides to built-in transform functions for coercing values to boolean and numbers.
  • Input aliases: allow us to rename an input property for use in templates by specifying an alias option in the decorator. While Angular generally recommends avoiding using aliases, they can come in handy in certain scenearios such as renaming properties or avoiding naming conflicts. the page.
  • Input with getters and setters: you may define an input property with a getter or setter to control how the value is accessed. This allows you to add additional logic when the value is set or retrieved. However, according to the documentation these are generally not recommended because of performance issues.

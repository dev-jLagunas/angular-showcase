Components Essentials

1. What are components?

Components are like the individual lego blocks we use to create full applications in Angular. This means we can build a tree of components which add up to create easily maintained and scalable apps.

2. How are components defined?

  • Components must have the following:
  • A @Component decorator with meta data and configuration
  • The HTML template for our user interface. This maybe inline or a link to separate
  • A CSS selector that will represent our actual component
  • The Typescript class with our behaviors, state, input, and data
  • Other important metada are our styles property and a standalone property (which is now recommended by the Angular team)

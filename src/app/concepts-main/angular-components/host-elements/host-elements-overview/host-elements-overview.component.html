1. What are "host" elements in Angular?

A host element is the actual HTML tag/element where a components template is rendered. In other words a host element is nothing more than the HTML tag used to display their respective components template. I am a dummy so for some reason this concept was rather annoying to grasp. Don't be like me, understand deeply!

It is important to grasp this concept of host elements because they provide the structure within which components function. These are the encapsulated template and logic of our reusable/modular components.

2. How and why do we bind to these host elements?

We can use these host elements to bind properties, attributes, and events directly to the components HTML tag. This is done using the "host" property in the component decorator. It's like adding functionality to the HTML tag itself. What does this actually mean?

  • It means- we can add properties, attributes, and events directly to the HTML tag inside the host property of the decorator.
  • It also means- we are configuring our component tag to behave in a certain way or have specific attributes once it is used in the template.
  • Using "host" property- using the "host" property to bind to the host element is the recommended method for modern Angular programming.
  • The HostBinding and HostListener decorators explained below are mainly kept for backwards compatibility as Angular recommends using the Host property explained above.

3. How do we work with multiple content placeholders?

Angular allows us to project multiple different elements into different ng-content placeholders based on a selector. Let's see how this works with an example below.

  • Child Component Class- now with multiple ng-content tags we use the select attritube to give them unique identifiers. Below you can see we have the select attributes equal to a name that reflects the content that will be projected.
  • Parent Component Class- using the same selector value we set inside the child component, we can use that custom selector to add the content we want to project into the child.
  • No matching selectors- if the child component has multiple ng-content tags with their select attributes properly set, but also tags with no select attribute, the 'plain' ng-content tag will capture all content that did not match any selector tags.
